# Contributing to NETSTALKER

Thank you for your interest in contributing to NETSTALKER! This document provides guidelines for contributing to the project.

## Code of Conduct

- Be respectful and professional
- Focus on ethical hacking and authorized testing
- Never promote illegal activities
- Help maintain a welcoming community

## How to Contribute

### Reporting Bugs

1. Check if the bug has already been reported in Issues
2. If not, create a new issue with:
   - Clear, descriptive title
   - Steps to reproduce
   - Expected vs actual behavior
   - Your environment (OS, Python version, etc.)

### Suggesting Enhancements

1. Open an issue describing:
   - The enhancement
   - Why it would be useful
   - How it should work

### Pull Requests

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/amazing-feature`
3. Make your changes
4. Follow the code style (see below)
5. Test thoroughly
6. Commit with clear messages
7. Push to your fork
8. Open a Pull Request

## Code Style

### Python Style

- Follow PEP 8
- Use 4 spaces for indentation
- Maximum line length: 100 characters
- Use docstrings for functions and classes

### Rose Pine Theme

- Use colors from `modules/theme.py`
- Maintain cyberpunk aesthetic
- Keep status indicators consistent

### Commit Messages

```
feat: Add new RustScan preset for wireless networks
fix: Correct NetExec hosts file generation
docs: Update README with new examples
style: Apply Rose Pine colors to new module
```

## Testing

- Test on Kali Linux and Ubuntu 22.04+
- Verify all tools still work
- Check output files are created correctly
- Test both CLI and interactive modes

## Documentation

- Update README.md for new features
- Add docstrings to new functions
- Include usage examples
- Update CLI reference if needed

## Legal Requirements

- Only contribute features for **authorized** security testing
- Do not add exploits or malicious code
- Ensure all contributions comply with the law
- Add appropriate disclaimers for risky features

## Questions?

Open an issue labeled "question" and we'll help you out!
